Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

de Casteljau’s algorithm gives us an algorithm to evaluate a point on a Bezier curve given endpoints and intermediate control points. The algorithm recursively linearly interpolates between adjacent points with a given parameter t, creating a new set of control points one fewer than before. The new control points are linearly interpolated again, repeating the process until a single point remains which is on the Bezier curve. Sweeping t from 0 to 1 traces out the full Bezier curve.
In this last image, the curve has been changed and the parameter t has been increased, moving the point further to the right.

Part 2: Bezier surfaces with separable 1D de Casteljau

de Casteljau's algorithm is naturally extended to 2D for a grid of n by n control points. First, we run n parallel instances of the 1D de Casteljau's algorithm along one axis using the same parameter t (now called u), each on n points. This gives us n different points on n different Bezier curves. Next, with a distinct t (now called v), we use the n determined points to find one more point on the Bezier curve determined by those points. Varying u and v from 0 to 1 traces out the full Bezier surface.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

Using the iteration example from the primer, we are able to grab pairs of vertices that form a triangle with the vertex the function is called on. For each pair of other vertices, we calculate the vector between those vertices and the vertex we called the function on. This lets us take a cross product, which gives the normal to the face. We sum up all the normals that we get from all the cross products and divide by the total of all norms. This gives us area weighting since the norm of the cross product is proportional to the area of the triangle. This gives a smoother shading result, seen below.

Part 6: Loop subdivision for mesh upsampling

For this part, we followed the suggested implementation route. The first two steps calculate new positions for existing vertices and edges for later. Then, we split all edges for our 4:1 split. However, in order to make all the new triangles be less skinny, we flip edges connecting new and old vertices. This makes a pair of skinny triangles less skinny. After this, the vertices are moved to their new locations. One issue was that we needed to go back into the previous parts and add markers for isNew to use in this part.
The cube ends up looking asymmetrical because the triangle mesh itself starts off asymmetrical. By splitting each edge once before upsampling, it causes all faces to look the same (X pattern), which means that the final cube is fully symmetrical. Alternatively, the edges can be flipped so that adjacent vertices have degree 0 and 3, alternating. This increases the number of axes of symmetry, but it is not as symmetrical as the split case, in a sense.
